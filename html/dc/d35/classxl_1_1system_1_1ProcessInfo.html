https://gist.github.com/nonowarn/770696

Make a sysctl() call to get the raw argument space of the process. The layout is documented in start.s, which is part of the Csu project. In summary, it looks like:

/------------—\ 0x00000000 : : : : |------------—|

argc
arg[0]
------------—

: : : : |------------—|

arg[argc - 1]
0
------------—
env[0]
------------—

: : : : |------------—|

env[n]
0

|------------—| <– Beginning of data returned by sysctl() is here.

argc
exec_path
:::::::::::::::
String area.

|------------—| <– Top of stack. : : : : ------------—/ 0xffffffff

Iterate through the '\0'-terminated strings and convert '\0' to ' ' until a string is found that has a '=' character in it (or there are no more strings in procargs). There is no way to deterministically know where the command arguments end and the environment strings start, which is why the '=' character is searched for as a heuristic.
