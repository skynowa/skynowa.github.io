std::size_t xl::db::mysql::IResult::fields ( ) const

number of columns in a result set / connection

Get the number of columns either from a pointer to a result set or to a connection handler.

You would use the connection handler if mysql_store_result() or mysql_use_result() returned NULL (and thus you have no result set pointer). In this case, you can call mysql_field_count() to determine whether mysql_store_result() should have produced a nonempty result. This enables the client program to take proper action without knowing whether the query was a SELECT (or SELECT-like) statement. The example shown here illustrates how this may be done.

number of rows in the result set / connection

mysql_num_rows() is intended for use with statements that return a result set, such as SELECT.

For statements such as INSERT, UPDATE, or DELETE, the number of affected rows can be obtained with mysql_affected_rows().
